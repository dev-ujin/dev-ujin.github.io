#### 1. 정보화사회의 정보보호
##### 정보사회의 특성
- 초고속 정보통신 기반구조를 바탕으로 **인터넷의 확산과 전자상거래**를 통하여 세계는 **단일 생활권**으로 압축됨
- 인터넷을 통한 신속한 정보교환으로 **재택근무가 증가**하고, 홈뱅킹, 사이버주식거래가 가능해졌음
- 정보전쟁 : 자동화된 **지휘통지시스템**과 해커, 바이러스를 이용하는 **사이버 전쟁 개념**이 복합된 것
   
##### 정보화 역기능
- 프라이버시 침해, 해커와 바이러스의 기승, 불법적인 위변조, 정보시스템의 파괴
    
##### 정보보호 개념
- 사전적 의미 : 정보의 수집, 가공, 저장, 검색, 송신, 수신 중에 발생하는 정보의 훼손, 변조, 유출 등을 방지하기 위한 관리적, 기술적 수단 또는 그러한 수단으로 이루어지는 행위
- **기밀성, 무결성, 가용성, 인증성 및 부인방지**를 보장하기 위하여 **기술적, 물리적, 관리적 보호 대책을 강구**하는 것
- 정보의 가용성과 보안 측면에서의 의미 : **정보의 활용**과 **정보의 통제** 사이에서 균형 감각을 갖는 행위
   
##### 정보보호 목표
1. `기밀성(Confidentiality)` : **오직 인가된 사람, 프로세스, 시스템만이 알 필요성에 근거해 시스템에 접근해야함**
  - 데이터의 비밀 엄수(secrecy)
  - 보안 기술 : 접근제어, 암호화 등
  - <r>전송중인 메시지가 공격자에게 노출되는 것에 대응 = 기밀성에 대한 설명</r>
2. `무결성 (Integrity)` : **정보의 내용이 불법적으로 생성, 변경, 삭제되지 않도록 보호해야함**
- 전력차단과 같은 악의 없는 행위가 원인이 될 수도 있음
- 보장 기술 : 접근제어, 메시지 인증 등
- 변경 탐지해 복구하는 기술 : 침입 탐지, 백업 등
3. `가용성 (Availability)` : **시스템이 지체 없이 동작하도록 하고, 합법적 사용자가 서비스 사용을 거절당하지 않도록 하는 것**
4. `인증성 (인증, Authentication)` : **사용자가 정말 그 사용자인지와 시스템에 도착한 자료가 정말 신뢰할 수 있는 출처에서 온 것인지 확인**
5. `책임추정성(책임성, Accountability)` : **개체의 행동을 유일하게 추적해서 찾아낼 수 있어야 함**

2. `물리적 보호대책` : 자연재해 대책, 출입통제, 시건(잠금)장치

#### 2. OSI 보안구조
##### OSI 보안구조 핵심
- 보안공격 : **정보의 안전성을 침해**하는 제반 행위
- 보안 메커니즘 : **보안 공격을 탐지, 예방**하거나 공격으로 인한 **침해를 복구하는 절차**
- 보안 서비스 : **보안을 강화하기 위한 처리 또는 통신 서비스**

##### 보안 공격
###### 기밀성을 위협하는 공격
- `스누핑(Snooping)` : 데이터에 대한 비인가 접근 또는 탈취(**가로채어 외부로 노출**)
- `트래픽분석(Traffic Analysis)` : 온라인 트래픽 분석을 통해 다른 형태의 정보를 얻음

###### 무결성을 위협하는 공격
- `변경(Modification)` : 메시지를 불법으로 수정하거나 순서를 바꿈  
- `가장(Masquerading)` : **한 개체가 다른 개체의 행세를 하는 것**  
- `재연(재전송, Replaying)` : **데이터를 보관하고 있다가 시간이 경과한 후에 재전송하여 인가되지 않은 사항에 접근하는 효과를 노리는 행위**  
- `부인(Repudiation)` : **송신자는 메시지를 보냈다는 것을, 수신자는 받았다는 것을 부인**
  - 부인방지(부인봉쇄, Nonrepudiation)

###### 가용성을 위협하는 공격
- `서비스 거부(Denial of Service)`

###### 소극적 공격과 적극적 공격
1. `소극적 공격` : 시스템 자원에는 영향을 끼치지 않고, **단지 정보를 획득**하는 것
 - 탐지보다 **예방**에 더 신경써야 함
2. `적극적 공격` : **시스템 자원을 변경**하거나 **작동에 영향**을 미치는 공격
  - 방어보다 **탐지**가 더 쉬움(공격자가 다양한 방법을 이용하기 때문)

#### 3. 보안용어
- `자산(Asset)` : **데이터** 혹은 데이터 소유자가 **가치를 부여한 실체**
- `취약점(Vulnerability)` : 공격자에게 열린 문을 제공할 수 있는 **SW, HW, 절차, 인력 상의 약점**, **위협의 이용대상**으로 **관리, 물리, 기술적 약점**
- `위협(Threat)` : **손실이나 손상의 원인**이 될 가능성을 제공하는 환경의 집합
  1. `가로채기(Interception)`
     - : 비인가된 당사자의 자산으로의 접근을 획득
     - **기밀성에 영향**을 줌
  2. `가로막음(Interruption)`
     - : 시스템 자원이
     - **가용성에 영향**을 줌
  3. `변조(Modification)`
     - : 비인가된 사용자가 접근하여 그 내용을 변경
     - **무결성에 영향**을 줌
  4. `위조(Fabrication)`
     - : 비인가된 당사자가 가짜 정보를 생성
     - **무결성에 영향**을 줌
- `위협주체(위협원, Threat agents)` : **취약점을 이용하는 존재**
- `위험(Risk)` : 위협주체가 취약점을 활용할 수 있는 가능성과 자산에 악영향을 미치는 결과
  - **위험 = 자산 * 위협 * 취약점**
- `노출(Exposure)` : 위협 주체로 인해서 **손실이 발생할 수 있는 경우**
- `대책/안전장치` : **취약점을 제거**하거나, 위협 주체가 취약점을 이용할 수 있는 가능성을 **감소시키기 위한 SW설정, HW장비, 절차**
- `다계층보안/심층 방어(Multi Layered Security/Defense in Depth)` : 여러 계층의 보안대책이나 대응수단을 구성하는 것
  - **한 가지 통제가 대응에 실패하더라도 전체 시스템을 위험에 빠뜨리지 않음**
  - **가장 최선의 보안 방법**
  - 보호, 탐지, 대응으로 이루어진 보안 접근법
- `직무상의 신의성실, 노력(Due Care, Due Diligence)`
  - Due : 특정 목적을 위하여 필요하거나 요구되는 적절하고 충분한 의무
  - Due Care : 특정 목적을 위하여 필요하거나 요구되는 적절하고 충분한 주의
  - Due Diligence : 특정 목적을 위하여 필요하거나 요구되는 적절하고 충분한 노력
- `사회공학(Social Engineering)` : **인간 상호의 깊은 신뢰**를 바탕으로 **사람들을 속**여 **정상적인 보안 절차를 깨뜨리기 위한 침입 수단**
- `시점별 통제(Control)`
  1. `예방통제(Preventive Control)`
     - : 사전에 위협과 취약점을 대처하는 통제
  2. `탐지통제(Detective Control)`
     - :  위협을 탐지하는 통제
  3. `교정통제(Corrective Control)` 
     - : 이미 탐지된 위협이나 취약점에 대처하거나, 감소시키는 통제

- **"보안은 불공정한 게임이다"** : 방어측은 모든 취약점을 다 알아야 하지만, 공격측은 하나만 알아도 됨
- **가장 약한 링크 원칙(Principle of Weakness Link)** : 보안은 가장 약한 링크보다 더 강할 수 없음
  - 어떤 하나의 제어수단의 실패가 전체 보안실패를 야기한다는 것을 의미
