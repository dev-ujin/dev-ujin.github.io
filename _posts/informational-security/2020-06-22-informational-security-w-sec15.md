# [1] UNIX 개념
_____
## 특징
- 대화식 운영체제 : 사용자와 운영체제간의 상호작용은 명령어 해석기인 **셸(Shell)**에 의해 이루어짐
- 멀티태스킹 : 여러 개의 명령어를 동시에 처리하는 방식
- 멀티유저 : 동시에 여러 사용자가 하나의 컴퓨터를 사용 가능. 보안이 중요해짐
- 호환성 및 이식성 : 하드웨어적인 구조에 독립적으로 응용프로그램을 작성할 수 있는 환경 제공
- 계층적 파일시스템 : 정보의 고유구조를 반영하도록 그룹화하는 기능으로 파일관리에 상당한 유연성 제공
- 뛰어난 통신 기능 : TCP/IP에 기반한 통신기능을 운영체제 자체에 내장하고 있으며, 프로세스 간을 연결하는 파이프 기능 제공
- 다양한 기능의 유틸리티와 개발환경 제공 : 수백여 개의 명령어와 유틸리티를 사용자 및 개발자에게 제공하여 도움을 줌

## 구성

### 커널(Kernel)
- 항상 메모리에 상주하여 CPU, 메인 메모리, 하드디스크 등의 하드웨어 자원을 제어
- 프로세스 스케줄링, 기억장치관리, 파일관리, 시스템호출 인터페이스, 입출력서비스 등의 기능을 사용자에게 제공
- 가장 하위 수준에서 하드웨어와 직접 관계함

### 셸(Shell)
- 운영체제와 사용자가 대화하기 위한 기반을 제공하는 중요한 프로그램
- 명령어 해석기
- 본 셸, C 셸, 콘 셸, 배시 셸 등 다양한 종류가 있음

### 파일시스템(File System)
- 디렉터리와 파일로 구성
- 디렉터리는 트리구조이며, 최상위 디렉토리는 루트임
- 구조
  - 부트 블록 
  - 슈퍼 블록 : 파일시스템의 요약정보와 사용하지 않고 있는 디스크 블록의 위치정보 가짐
  - i-node : 파일에 대한 정보를 기억하는 120byte의 고정된 크기의 구조체
  - 데이터 블록




# [2] UNIX 기본 사용법
_____
## 1. 디렉터리와 파일 정보 출력(ls)
### 옵션
  - -l : 리스트 형태로 정보를 자세히 보여줌
  - -a : 도트(.)파일을 포함하여 디렉터리 내 모든 디렉터리와 파일을 보여줌
  - -R : 하위 디렉터리에 있는 내용까지 보여줌
  - -F : 디렉터리인지, 어떤 종류의 파일인지 보여줌(/ : 디렉터리, * : 실행파일, @ : 심볼릭링크)
  - -i : 디렉터리 및 파일에 지정된 i-node번호를 보여줌
  - 여러 문자를 연속적으로 사용하여 옵션 기능을 복합적으로 사용할 수 있음

### 파일 유형
- &#45; : 정규파일
- d : 디렉터리 파일
- l : 심볼릭 링크 파일
- b : 블록 장치 특수 파일
- c : 문자 장치 특수 파일
- p : 파이프, 프로세스 간 통신에 사용되는 특수 파일
- s : 소켓, 네트워크 통신에 사용되는 특수 파일

### 접근 권한
- 대상
  - user : 파일을 만든 소유주
  - group : user가 속한 그룹의 사용자
  - other : 기타 사용자

- 종류

- 디폴트 설정
  - 일반 파일 : 666
  - 디렉터리 : 777

### 하드 링크수(링크 카운터)
- 동일한 i-node를 참조하는 파일의 수

## 2. 접근권한 변경(chmod)

### 옵션
- -R : 하위 디렉터리와 파일의 권한까지 변경
- permission : 기호나 8진수로 접근권한을 지정

### 접근권한을 기호로 기술하는 방법
- 대상 : u(user), g(group), o(other), a(all)
- 연산자 : +(추가), -(제거), =(지정)
- 접근권한 : r(읽기), w(쓰기), x(실행)

### 접근권한을 숫자로 기술하는 방법
- r(읽기) = 4, w(쓰기) = 2, x(실행) = 1

### 소유권/그룹 변경
- 소유권 변경 : chown
- 소유그룹 변경 : chgrp
- 옵션
  - -R : 하위 디렉터리와 디렉터리 하위의 모든 파일의 소유주를 변경
  - -h : 심볼릭 링크 파일 자체의 소유주나 그룹을 변경

## 3. 프로세스 생성과 종료(fork, exec)

- 새로운 프로세스가 필요할 땐 기존의 프로세스를 복제해서 사용
- 부팅 시간 동안 PID가 0인 swapper 프로세스가 PID 1인 init 프로세스와 PID 2인 pagedaemon 프로세스를 만듦

### 규칙
- PID가 0인 프로세스를 제외한 모든 프로세스는 실행 중에 부모 프로세스를 가짐
- 프로세스 종료 시 자신을 생성한 부모 프로세스에게 자신의 종료를 알림
- 부모 프로세스가 자식의 종료를 확인할 때까지 자식은 좀비 프로세스로 남음





# [3] UNIX 시스템 관리
_____
## 1. 시스템 시작(부팅)

### 런 레벨(Run Level)
- 시스템의 운영 상태를 숫자 혹은 문자로 표현한 것
- init은 런 레벨을 옮겨 다니며 각 레벨마다 역할을 수행(명령어 : init 런레벨)

## 2. 시스템 종료

### 주의사항
- 접속 중인 사용자에게 시스템의 종료를 공지하여 작업을 마무리하도록 해야 함
- 운영 중인 프로세스를 안전하게 종료해야 함
- 하드디스크를 갱신하여 파일시스템의 무결성을 유지해야 함
- **shutdown** 명령은 시스템을 안전하게 종료할 때 사용하며, 시스템의 런 레벨을 변경할 때도 사용함 

### 하드디스크 동기화(sync)
- 시스템 종료 중에 버퍼에 있는 데이터가 하드디스크에 반영되도록 해야 함

## 3. 파일시스템 연결(mount)
- 보조기억장치에 설치된 파일시스템을 UNIX가 이닉하도록 특정 디렉터리에 논리적으로 연결시킴

### 옵션
- -a : /etc/fstab 파일에 정의된 모든 파일시스템을 마운트함

## 4. 파일시스템 연결 해제(umount)
- 이전에 마운트 된 파일시스템의 연결을 해제
