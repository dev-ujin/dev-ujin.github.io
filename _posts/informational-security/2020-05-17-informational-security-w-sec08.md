> 알기사 교재 - section 08. 접근통제 개요

# [1] 접근통제(접근제어, Access Control) 개념
_____
- 접근통제 : 적절하지 못한 접근을 차단하기 위한 방법과 규칙의 집합체
  - 비인가된 사용자의 정보자원 사용의 방지
  - 인가된 사용자가 비인가된 방식으로 정보 자산을 접근하는 행위 방지
  - 인가된 사용자가 인가된 방식으로 정보 자산을 실수 혹은 의도적으로 훼손하는 행위 방지




# [2] 접근통제 구성
_____
1. `주체(Subject)`
    - 객체나 객체 내의 데이터에 대한 **접근을 요청하는 능동적인 개체**
    - 사용자, 프로그램, 프로세스가 해당될 수 있음
    - **행위자**라고 불림
2. `객체(Object)`
    - **접근대상**이 될 수동적인 **개체 혹은 행위가 일어날 아이템**
    - 컴퓨터, 데이터베이스, 파일, 컴퓨터 프로그램, 디렉터리가 해당될 수 있음
    - **제공자**라고 불림
3. `접근(Access)`
   - **주체와 객체 사이의 정보 흐름**
   - **읽고 만들고 삭제하거나 수정**하는 등의 행위 




# [3] 접근통제 절차
_____
1. `식별(Identification)` : **본인이 누구라는 것을 시스템에 밝히는 것**
   - 사용자명, 계정번호, 메모리카드
2. `인증(Authentication)` : 본인임을 주장하는 사용자가 그 본인이 맞다고 **시스템이 인정해주는 것**
   - 패스워드, PIN, 토큰, 스마트카드, 생체인증
3. `인가(Authorization)` : 인증된 주체에게 접근을 허용하고 특정 업무를 수행할 **권리를 부여하는 과정**
   - `알 필요성(Need-to-Know)` : **주체에게 어떤 정보가 유용할지** 여부와 관계가 있는 공인된 형식상의 접근수준
   - 접근제어목록(ACL), 보안 등급
4. `책임추적성(Accountability)` : 사용자의 이용을 추적하고 그의 행동에 대해 기록하고 추적하는 활동
   - 최근에 3단계에 덧붙여 추가됨
   - 사전 침입 의도를 감소시킴
   - 하나의 계정을 여러 명이 공유할 경우에는 책임추적이 어려움
 



# [4] 접근통제 요구사항
_____
1. `입력의 신뢰성(Input Reliability)` : 시스템이 요구하는 수준의 보안 서비스를 제공하는데 **충분한 사용자 인증 기능**이 전단에 존재함을 전제로 함
2. `최소 권한 부여(Least Privilege)` : 사용자가 작업을 수행하는데 필요한 **최소한의 자원**과 자원에 대한 **최소한의 접근 권한**을 부여하도록 구현되어야 함
   - 사용자가 의도적으로 혹은 실수로 자원에 대해 입힐 수 있는 **손상을 최소화하기 위해 중요**한 요구사항
3. `직무 분리(Separation of Duty)` : 시스템 **기능의 단계를 다수의 개인들에게 나누어야 함**
   - **한 개인이 전체 업무를 파괴하는 사고를 방지하기 위해**
