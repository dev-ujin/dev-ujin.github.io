> 알기사 교재 - section 12. 보안 운영체제

# [1] 보안 운영체제
_____
### 개념
- 운영체제 내에 보안기능을 통합시킨 보안커널을 추가로 이식한 운영체제

### 보호 대상
- 메모리
- 보조기억장치 상의 파일 또는 데이터의 집합
- 메모리상에서 실행 중인 프로그램
- 파일들의 디렉터리
- 하드웨어 장치

### 보호 방법
- 물리적 분리(Physical Separation) : 사용자별로 별도의 장비만 사용하도록 제한하는 방법
- 시간적 분리(Temporal Separation) : 프로세스가 동일 시간에 하나씩만 실행되도록 하는 방법. 운영체제의 일을 단순화 시킴
- 논리적 분리(Logical Separation) : 각 프로세스에 논리적인 구역을 지정하는 방법
- 암호적 분리(Cryptographic Separation) : 내부에서 사용되는 정보를 외부에서는 알 수 없도록 암호화하는 방법
- 구현 복잡도가 증가하는 순서

### 보안 기능
1. 사용자 식별 및 인증
2. 임의적/강제적 접근 통제
   - 임의적 접근통제(DAC)는 신분에 근거하여 객체에 대한 접근을 제한하기 때문에 신분기반정책이라고도 함
   - 강제적 접근통제(MAC)는 객체의 비밀등급과 주체가 갖는 권한에 근거하여 객체에 대한 접근을 제한하기 때문에 규칙기반정책이라고도 함
3. 객체 재사용(Object Reuse) 보호 : 재할당되는 모든 기억장치 공간을 깨끗하게 지워야 함
4. 완전한 조정 : 모든 접근을 통제하는 완전한 조정을 수행하여야 함
5. 신뢰 경로(Trusted Path) : 안전한 통신을 위한 안전한 경로를 제공할 수 있어야 함
6. 감사 및 감사 기록 축소 : 감사기록부(Audig Log)에 반드시 기록되어야 하고 감사 기록부는 명백하게 보호되어야 함

# [2] 신뢰 컴퓨팅 기반(TCB, Trusted Computing Base)
_____
### 개념
- 하나의 컴퓨터 시스템(하드웨어, 소프트웨어, 펌웨어) 내의 모든 보호 메커니즘의 총체
- 시스템과의 조화를 통하여 보안 정책을 적용할 책임을 가짐
- 신뢰된 경로(Trusted Path) : 사용자, 프로그램 그리고 커널과의 통신채널
- 보안 경계(Security Perimeter) : 시스템의 구성요소 중 신뢰/비신뢰 구성요소의 경계로 가상의 경계선

### TCB의 4가지 모니터 기능
1. 프로세스 활성화
2. 실행 도메인 변경
3. 메모리 보호
4. 입출력 연산

# [3] 참조모니터(Reference Monitor)
_____
### 개념
- 주체의 객체에 대한 모든 접근통제를 담당하는 추상머신
- 승인되지 않은 접근이나 변경으로부터 객체를 보호하기 위해 객체에 대한 주체의 모든 접근통제를 중재하는 개념

### 규칙
1. 반드시 부정조작이 없어야 함(Must be tamperprrof)
2. 항상 무시되지 않고 호출되어야 함(Always invoked)
3. 모든 동작을 항상 분석과 테스트를 통해 확인할 수 있어야 함(Must be Verifiable)

- 위반하는 경우는 주체가 참조모니터를 거치지 않고 객체에 직접 접근하는 경우
- 보안커널 데이터베이스(SKDB, Security Kernel Data Base)를 참조하여 객체에 대한 접근허가 여부를 결정하여야 함

# [4] 보안커널(Security Kernel)
_____
### 개념
- TCB 내에 있는 하드웨어, 소프트웨어, 펌웨어로 구성됨
- 참조 모니터 개념을 구현하고 집행
- 주체와 객체 사이의 모든 접근과 기능을 중재

### 요구사항
1. 참조 모니터 개념을 수행하는 프로세스를 위해 분리가 제공되고 프로세서는 변조를 방지할 수 있어야 함
2. 모든 접근 시도에 대하여 실행되어야 하고, 이것을 우회하는 것이 불가능해야 함
3. 보안 커널은 완전하고 포괄적인 방식으로 충분히 시험되고 확인되어야 함

# [5] 신뢰 플랫폼 모듈(TPM)
_____
### 개념
- 신뢰 컴퓨팅 그룹(Trusted Computing Group)이라는 산업체 컨소시엄에 의해 표준화된 개념
- 신뢰 컴퓨팅에서 핵심이 되는 하드웨엄 모듈

### 특징
- 하드웨어칩으로 구현하는 것이 일반적이지만, 소프트웨어로 구현하기도 함
- 물리적인 도난의 경우에도 정보의 노출이 용이하지 않다는 장점을 가짐
- 관련 연산 : 암호화 키의 생성과 저장, 패스워드의 저장, 무결성의 검증, 디지털 인즈서 관련 신뢰 연산

### 기본 서비스
- 인증된 부트 서비스 : 전체 운영체제를 단계적으로 부팅하고, 운영체제의 각 부분이 승인된 버전임을 보장
- 인증 서비스 : 사설키(private key)를 사용하여 설정 정보에 서명함으로써 디지털 인증을 만들 수 있음
- 암호화 서비스 : 특정 기계가 특정 설정으로 되어 있을 때만 그 기계에서 데이터의 복호화를 수행하는 방식으로 데이터의 암호화를 지원

### 구성 요소
1. I/O : 모든 명령어는 I/O 구성요소를 통해 들어오고 나가며, 다른 TPM 구성요소와의 통신을 제공
2. 암호화 보조 프로세서 : 암호화 및 관련 처리
3. 키 생성 : RSA 공유/사설키 쌍과 대칭키를 만듦
4. HMAC 엔진 : 다양한 인증 프로토콜에서 사용됨
5. 무작위 숫자 발생기(RNG) : 키 생성, 전자서명의 무작위 값, 임시수(비표)를 생성할 때 사용
6. SHA-1 엔진 : 전자서명과 HMAC 알고리즘에서 사용되는 SHA 알고리즘을 구현
7. 전원 탐지 : TPM 전원 상태와 플랫폼 전원 상태를 관리
8. Opt-in : TPM을 활성 또는 비활성화할 수 있는 안전한 메커니즘 제공
9. 실행 엔진 : TPM 명령어를 수행할 프로그램 코드를 실행
10. 비휘발성 메모리 : TPM을 위한 신원, 상태 인자를 저장
11. 휘발성 메모리 : 임시 저장 장치로 현재 TPM 상태. 암호화 키, 세션 정보를 저장
