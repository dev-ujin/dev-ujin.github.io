> 2020 알기사 정보보안기사 `section 02. 암호학 개요`를 공부한 내용을 바탕으로 작성하였습니다.

##### 대칭키 암호(symmetric cryptography)와 비대칭키 암호(asymmetric cryptography)
- `대칭키 암호` : 암호화할 때 사용하는 키와 복호화할 때 사용하는 키가 동일한 암호 알고리즘 방식.
- `비대칭키 암호` : 암호화할 때 사용하는 키와 복호화할 때 사용하는 키가 다른 암호 알고리즘 방식.

##### 하이브리드 암호 시스템(hybrid cryptosystem)
- `하이브리드 암호 시스템` : 대칭키 암호와 공개키 암호의 장점을 조합한 암호 방식.

##### 일방향 해시함수(one-way hash function)
- 해시값 : 일방향 해시함수를 사용하여 계산한 값.
- 문서의 **무결성(완전성)**을 확인할 수 있음.

##### 메시지 인증코드(message authentication code)
- 메시지가 생각했던 통신 상대로부터 온 것임을 확인할 수 있음.
- **무결성**(메시지가 전송 도중에 변경되지 않음)과 **인증**(생각했던 통신 상대로부터 온 것임을 확인)을 제공.

##### 전자서명
- **무결성**을 확인하고, **인증**과 **부인방지**를 하기 위한 암호 기술.
- 거짓행세(spoofing), 변경, 부인이라는 위협을 방지할 수 있음.

##### 의사난수 생성기(PRNG, pseudo random number generator)
- `의사난수 생성기` : 난수열을 생성하는 알고리즘.
- 난수는 키 생성이라는 매우 중요한 역할을 담당.
