# [1] 접근통제의 모델
## 1. 강제적 접근통제(MAC, Mandatory Access Control)

### 개념
- 보안 관리자가 취급인가를 허용한 개체만 접근할 수 있도록 강제적으로 통제
- 보안 레이블(시스템 자원이 얼마나 중요한지)과 보안 허가증(개체가 자원에 접근할 수 있는지)을 비교하는 것에 기반

### 문제점
- 제한적 사용자 기능
- 많은 비용 소모
- 사용자 친화적이지 않음
- 모든 접근에 대해 레이블링을 정의하고 보안 정책을 확인해야하기 때문에 성능저하가 우려됨 

## 2. 임의적 접근통제(DAC, Discretionary Access Control)

### 개념
- 객체의 소유자가 주체가 속한 그룹의 신원에 근거하여 객체에 대한 접근을 제한하는 방법
- 세부적으로 신원 기반 접근통제, 사용자 기반 접근통제, 혼합 방식 접근통제 등으로 나뉨

### 기법
1. 접근제어 행렬(Access Control Matrix) : 행렬을 이용하여 주체, 객체, 접근권한의 관계를 기술하는 방법
    - 주체를 행(Row), 객체를 열(Column)으로 구성하고 셀에 접근권한을 기록
    - 주체와 객체의 수가 많아질 경우 행렬의 크기가 커져 관리가 어려워짐
    - 비어있는 셀의 수가 많아져 공간의 비효율성 문제가 있음
2. 자격 목록(권한 리스트, Capability List/Tickets/Table) : 한 주체가 갖는 자격들의 리스트
    - 주체의 관점에서 접근권한을 부여
    - 컨텐츠의 보안성이 보장받지 못하는 분산환경에서 사용하기에 적합
    - 예 : 커버로스
3. 접근제어 목록(ACLs, Access Control Lists) : 객체에 대한 접근권한을 갖고 있는 주체들의 모임을 나타낸 것
    - 객체의 관점에서 접근권한을 부여

### 장점
- 객체별로 세분화된 접근제어가 가능
- 객체의 소유자가 객체에 대해 다른 주체에게 임의적으로 접근제어가 가능해 매우 유연한 접근 제어 서비스를 제공할 수 있음

### 단점
- 시스템 전체 차원의 일관성 있는 접근 제어가 부족할 수 있음
- 바이러스, 웜, 루트킷, 트로이목마 공격에 취약

## 3. 역할기반 접근통제(RBAC, Role Based Access Control)

### 개념
- 중앙 관리자에 의해 사용자의 역할에 기반을 두고 접근을 통제하는 모델
- 역할은 사용자 집합과 권한 집합의 매개채
- MAC과 DAC의 단점을 보완
- 주체의 인사이동이 잦은 조직에 적합

### 장점
- DAC에 비해 유연성은 떨어지지만 관리자에 의해 전체 시스템에 일관성 있는 접근제어가 용이
- 역할 수행에 반드시 필요한 접근권한만 부여함으로써 최소 권한(Least Privilege) 부여 원칙을 충족시키기가 용이
- 역할을 명확하게 분리하고 역할에 따른 접근권한을 명확하게 분리함으로써 직부 분리(Separation of Duty)원칙을 충족시키기도 용이


# [2] 보안 모델

## 1. 벨라파듈라 모델(BLP, Bell-LaPadula Confidentiality Model)

### 개념
- 기밀성 중심
- 상태머신 모델, 정보흐름 모델, MAC 모델
- 최초의 수학적 모델

### 보안 규칙
- No read up(단순 보안 속성(Simple Security Property)) : 주체는 갖거나 낮은 보안 수준의 객체만 읽을 수 있음
- No write down(*속성(Star Property)) : 주체는 같거나 높은 보안 수준의 객체에만 쓸 수 있음
- 특수 *속성 규칙(Strong Star Property Rule) : 주체가 객체에 읽기, 쓰기를 하기 위해서는 보안 허가와 보안 분류가 동일해야 함

### 장점
- 기밀성 유지에 매우 강한 특성을 가짐

### 단점
- 보안 단계가 다른 두 사용자가 공유 메모리 또는 네트워크 채널과 같은 자원을 공유하고 있을 경우 기밀성이 훼손될 수 있음
- 무결성 유지에는 문제가 될 수 있음

## 2. 비바 무결성 모델(Biba Integrity Model)

### 개념
- 무결성 중심
- 상태머신 모델(State Machine Models)
- 최초의 수학적 모결성 모델

### 보안 규칙
- 단순 무결성(Simple Integrity) : 객체의 무결성 수준이 주체의 무결성 수준보다 우세할 때 주체는 객체를 읽을 수 있음
- 무결성 제한(0Integrity Confinement, Star Integrity) : 주체의 무결성 수준이 객체의 무결성 수준보다 우세할 때 주체는 객체를 변경할 수 있음
- 호출 속성(Invocation Property) : 주체는 보다 높은 무결성을 갖는 주체에게 서비스를 요청할 수 없음 
- 무결성 모델의 목적 세 가지 중 한 가지만 만족
    1. 허가되지 않은 사용자로부터의 수정을 예방

## 3. 클락-윌슨 무결성 모델(Clark-Wilson Integrity Model)

### 개념
- 좀 더 정교하고 실제적인 무결성 모델

### 접근 통제 원칙
- 정확한 트랜잭션(Well Formed Transaction)
   - TP를 이용하여 CDI를 수정하는 것
   - 데이터를 하나의 일치 상태에서 다른 일치 상태로 변경하는 일련의 동작
- 무결성 모델의 목적 세 가지를 모두 만족
    1. 허가되지 않은 사용자로부터의 수정을 예방
    2. 허가된 사용자의 부적절한 수정을 예방
    3. 내부 및 외부 일치성을 유지(정확한 트랜잭션)

## 4. 만리장성 모델(Chinese Wall Model, Brewer-Nash Model)

### 개념
- 이해 충돌을 방지하기 위해 만리장성을 이용하자는 것
- 사용자의 이전 동작에 따라 변화할 수 있는 접근 통제를 제공
- 정보 흐름 모델
