# 🔥 실패한 접근법
## [1] 무식한 방법
문제를 그대로 직역하는 방법이다. for문을 돌면서 각 마을에 경기장을 세웠다고 가정하고 야구팬들과 경기장 사이의 거리의 합을 구해 어느 마을에서 최솟값을 가지는지 확인한다. 하지만 N의 범위가 1 < N <= 100,000이기 때문에 N<sup>2</sup> = 10,000,000,000이므로 1초를 한-참 초과한다.

## [2] 수식으로 접근해볼까?
일반화에 앞서서 예시에 먼저 접근해보기로 했다. 마을의 위치가 (1, 2, 5, 7, 8)로 주어졌고, 각 마을의 팬들의 수가 (3, 2, 4, 3, 1)로 주어졌다. 경기장을 세울 x축의 좌표를 x라고 하고 거리의 총합을 d라 하면 d = |3(x-1)| + |2(x-2)| + |4(x-5)| + |3(x-7)| + |(x-8)|이 된다. 자...그럼 x는 (1, 2, 5, 7, 8)중에 하나의 값을 가지니까 이 중에 어떤 값을 취해야 d가 최소가 될까... 도저히 모르겠어서 이건 아닌가보다...하고 패쓰! 혹시 이렇게 접근해서 해결하신 분이 있다면 저도 알려주세요...😢

# 🔥 해결방법
예시로 주어진 팬들의 수 (3, 2, 4, 3, 1)에서 '가운데 4를 기준으로 좌편이 3 + 2 = 5, 우편이 3 + 1 = 4니까 가운데보다 살짝 왼쪽에다 지으면 되겠다. 근데 어차피 경기장은 입력으로 주어진 마을 중에 하나에 지어야 하니까 결국 4명의 팬이 있는 마을에 경기장을 지어야겠구나.'라는 생각이 들었다. 이를 일반화하면 **'좌편과 우편을 똑같이 나눌 수 있는 값, 즉 중간값을 구한 다음, 그 값이 속하는 마을에 경기장을 세운다.'**이다. `O(n)`으로 풀린다!

## 알고리즘 풀이
1. 마을의 위치는 villages[]에 저장하고, 각 마을의 팬들의 수는 fans[]에 저장한다.
2. 각 마을의 팬들의 합을 구해 그 값의 중간값을 구한다.
3. 반복문을 실행해 fans[]을 차례대로 temp_sum에 더해나간다.
4. 더해나가다가 temp_sum이 중간값을 초과하면 villages[]를 통해 해당 마을의 위치를 반환한다.

## 주의
- 중간값은 소숫점까지 표현해야한다!

```cpp
...
while(temp_sum <= ((float)fan_sum / 2)) {
    temp_sum += fans[index++];
}
...
```
fan_sum이 int이기 때문에 중간값이 소숫점까지 표현되려면 형변환을 시켜주어야 한다. 문제에서 제시한 예시에서는 드러나지 않는 예외사항인데...마을의 위치는 (1, 2, 5, 7, 8)로 그대로 두되 팬들의 수를 (3, 2, 1, 3, 4)로 살짝 바꿔서 생각해보면 알 수 있다. 팬들의 수의 총합은 13으로 같다. 만약 형변환을 하지 않으면 중간값은 6으로 5에 위치한 마을에 경기장을 세워야 할 것이다. 하지만 형변환을 하여 소숫점까지 표현한 중간값은 6.5로 7에 위치한 마을에 경기장을 세워야 할 것이다. 이 경우에 정답은 7에 위치한 마을이다.
