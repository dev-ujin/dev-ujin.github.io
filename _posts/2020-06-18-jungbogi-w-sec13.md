# [1] 악성 소프트웨어(악성 코드)

### 개념
- 의도적으로 컴퓨터의 보안속성을 침해할 목적으로 작성된 프로그램

### 분류
1. 독립형과 기생형
   - 독립형 : 시스템 프로그램 없이 독립적으로 존재할 수 있음(예 : 웜, 좀비 바이러스 등)
   - 기생형 : 시스템 프로그램 없이는 독립적으로 존재할 수 없음(예 : 바이러스, 논리폭탄, 백도어 등)
2. 바이러스성과 비바이러스성
   - 바이러스성 : 자기복제를 하는 경우(예 : 웜, 바이러스 등)
   - 비바이러스성 : 자기복제를 하지 못하는 경우(예 : 트로이목마, 백도어 등)

## 1. 바이러스(Virus)

### 개념
- 다른 프로그램을 변형시키도록 하여 감염시키는 프로그램의 단편

### 세대별 분류
1. 제1세대 원시형 바이러스(Primitive Virus)
   - 코드의 변형이나 변화 없이 고정된 크기를 가지며, 주로 기억장소에 상주해서 부트 영역이나 파일을 감염시킴
   - 프로그램 구조가 단순하고 분석이 상대적으로 쉬움
2. 제2세대 암호화 바이러스(Encryption Virus)
   - 백신 프로그램의 진단을 피하기 위해 바이러스 프로그램의 일부 혹은 대부분을 암호화시켜 저장
3. 제3세대 은폐형 바이러스(Stealth Virus)
   - 백신 프로그램이 감연된 부분을 읽으려고 할 때 감염되기 전의 내용을 보여줘 바이러스가 없는 것처럼 속임
4. 제4세대 갑옷형 바이러스(Armour Virus)
   - 백신 프로그램 제작자에게 공격의 화살을 돌려 바이러스 분석을 어렵게 하고 백신 프로그램 개발을 지연시킴
5. 제5세대 매크로 바이러스(Macro Virus)
   - VBS(Visual Basic Script)를 지원하는 다양한 프로그램에서 활동하기 때문에 현재 등장하고 있는 바이러스 중에서 가장 높은 비중을 차지
   - 플랫폼과 무관하게 사용됨
   - 문서를 감염시키고 코드의 실행부분은 감염시키지 않음

### 방지책
- 안티 바이러스 필터링 방법
  - Signature Scanning : 특정 바이러스만이 가진 유일한 형태의 signature를 찾아내는 방법
  - Behavioral Virus Scanning : 바이러스가 수행 중에 어떤 행동을 보이는지를 추적하는 방법
- 예방
  - 신뢰성 있는 업체에서 구입한 상업용 소프트웨어를 사용
  - 안전하다고 생각될 때에만 첨부파일을 열어야 함
  - 바이러스 스캐너를 업데이트 하고, 이를 이용해 정기적으로 검사해야 함
  - Windows Script Host, ActiveX, VBScrip, JavaScript는 비활성화
  - 보안 정책을 수립하고 교육을 이행 

## 2. 웜(Worm)

### 개념
- 자신을 복제하여 네트워크 연결을 통해서 컴퓨터에서 컴퓨터로 그 복제본을 전송
- 다른 시스템에 직접적인 영향을 미치지 않는다는 점에서 트로이목마와 구분되고, 다른 프로그램에 기생하지 않는다는 점에서 바이러스와 구분됨

### 확산 모델
- 감염된 호스트는 각각 두 개의 호스트를 감염시키기 때문에 감염된 호스트의 수는 지수적으로 증가함
- 시간이 경과하면서 감염된 호스트는 이미 감염된 호스트를 공격하는데 시간을 허비하기 때문에 감염 속도가 줄어듦

### 대응책
- 웜 확산이 이뤄지면 네트워크 활동이 활발해지기 때문에 네트워크 사용량을 모니터링함

## 3. 트로이목마(Trojan horse)

### 개념
- 자신의 실체를 드러내지 않으면서 마치 다른 프로그램인 것처럼 가장하여 활동하는 프로그램
- 트로이목마가 포함된 프로그램이 실행되는 순간, 공격자는 시스템을 통제할 수 있는 권한을 갖게 됨

### 기능
- 원격 조정
- 패스워드 가로채기
- 키보드 입력 가로채기
- 시스템 파일 파괴

## 4. 기타 악성 소프트웨어
1. 논리폭탄 : 조건이 중촉되면 트리거가 작동
2. 백도어(트랩 도어) : 기능에 허가받지 않은 접근을 허용하는 프로그램의 변형
3. 모바일 코드 : 스크립트, 매크로나 다른 이동성 명령어 같은 소프트웨어
4. Exploit : 하나 혹은 여러 개의 취약점을 노리는 코드
5. Downloaders : 공격을 받는 컴퓨터에 다른 아이템을 설치하는 프로그램. 보통 전자우편을 통해 전달됨
6. Auto-Rooter : 악성 해커 도구로 새로운 시스템에 원격으로 침입할 때 사용하는 툴
7. KIT(Virus Generator) : 바이러스를 자동으로 생성하는 도구모음
8. 스페어 프로그램 : 원하지 않는 대량의 전자우편물을 보내는 데 사용됨
9. 플러더(Flooders/DoS Client) : 네트워크 컴퓨터 시스템에 대량의 자료를 보내어 서비스 거부 공격을 감행하는 데 사용되는 코드
10. Keyloggers : 피해를 입은 시스템의 키 입력을 갈취
11. Rootkit : 컴퓨터 시스템의 침입후 루트 수준의 접근허락을 얻기 위해 사용하는 해커 도구모음
12. 공격 킷(Attack Kit) : 다양한 번식방법과 Payload 기술을 사용하는 새로운 악성코드를 자동으로 만들어주는 툴의 모음
13. 좀비(Zombie) : 감염된 컴퓨터에서 활성화되는 프로그램으로 다른 컴퓨터에 대한 공격을 시작하는 데 사용됨
14. 스파이웨어(Spyware) : 컴퓨터에서 정보를 수집해서 다른 시스템으로 전송하는 소프트웨어
15. 애드웨어 : 소프트웨어에 내장된 광고, 감염시 팝업 광고가 뜨거나 브라우저가 광고 사이트로 연결됨
16. 크라임웨어(Crimeware) : 온라인을 통해 불법적인 행동(범죄)을 하기 위해 만들어진 프로그램
17. 브라우저 하이재커(Browser Hijacker) : 브라우저를 하이재킹하여 홈페이지와 검색 페이지, 툴바를 통제하고 조작하는 프로그램
18. 다이얼러(Dialers) : 모뎀이 특정 번호로 연결되도록 하여 전화를 걸 때마다 공격자가 수익을 얻게 만드는 프로그램
19. 조크(Joke) : 실제 바이러스는 아니지만 사용자에게 심리적인 위협이나 불안을 조장하는 프로그램
20. Hoax : 남을 속이거나 장난을 목적으로 퍼트리는 가짜 바이러스로, 일반적으로 허위 바이러스 경고 메일형태

# [2] 인터넷 활용 보안
## 1. 웹브라우저 보안

### 개념
- 여러 유형의 사이트를 관리하기 위해 다양한 보안영역을 지원하고 각 영역에 대한 보안 요구사항을 사용자가 정의할 수 있도록 함

### 사용 가능한 영역
1. 인터넷 : 다른 세 가지 영역에 포함되지 않은 웹사이트를 위한 영역(디폴트 : 약간 높음)
2. 로컬 인트라넷 : 사용자의 컴퓨터와 네트워크(인트라넷)상에 있는 웹사이트를 위한 영역(디폴트 : 낮음)
3. 신뢰할 수 있는 사이트 : 사용자가 안전한 컨텐츠를 갖고 있다고 신뢰하는 웹사이트를 위한 영역(디폴트 : 낮음(보통))
4. 제한된 사이트 : 사용자가 신뢰할 수 없거나 보안 페이지로 설정할 수 없는 웹사이트 영역(디폴트 : 높음)

## 2. 쿠키(Cookie)

### 개념
- 사용자가 웹 사이트를 방문할 때 생기는 4KB 이하의 파일로, 사용자와 웹 사이트를 연결해주는 정보가 담겨 있어 나중에 클라이언트가 해당 사이트를 접속하려고 할 때 쿠키의 내용을 이용하여 클라이언트의 신분을 알 수 있게 해줌

### 구조

```java
Set-Cookie: name=value; expires=[Date]; domain=[Domain]; path=[Path]; [secure]
```
- 유효기간 : 기본적으로는 브라우저가 종료될 때까지 쿠키의 데이터를 사용할 수 있지만, 유효 기간을 지정하면 브라우저가 종료되어도 지정한 기간 동안은 쿠키의 데이터를 읽고 쓸 수 있음
- 패스 : 쿠키 데이터를 생성한 웹 페이지에서만 그 데이터를 읽을 수 있지만, Path 항목을 지정해주면 해당 Path 이하에서는 그 쿠키 데이터를 공유할 수 있음
- 도메인 : 패스 속성을 확장한 것으로, 도메인 단위에서 쿠키 데이터를 읽고 쓰는 권한을 설정할 수 있음

### HTTP 쿠키 관련 보안 속성
- httpOnly 속성 : 클라이언트에서 스크립트를 통해 해당 쿠키에 접근하는 것을 차단해주는 속성(XSS에 대응)
- secure 속성 : 클라이언트에서 HTTPS통신일 경우에만 해당 쿠키를 전송하고 HTTP 통신일 경우에는 전송하지 않는 속성(기밀성 보장)

### 보안 취약점
1. XSS(Cross-Site Scripting) 공격 : 자바스크립트에서 document.cookie라는 명령어는 사이트에서 쿠키값을 사용할 수 있도록 하는데, 공격자들이 이를 유출하기 위한 목적으로 사용함
2. 스니핑(Sniffing) 공격을 이용 : 쿠키값을 따로 암호화하지 않고 전송한다면 네트워크 스니핑 공격을 통해 쿠키값을 탈취할 수 있음
3. 공용 PC에서 쿠키값 유출 : 공용 PC의 하드디스크에 저장된 쿠키 정보는 쉽게 얻을 수 있음

## 3. 세션(Session)

### 개념
- 개별 클라이언트 상태정보를 서버에 저장하는 기술
- 세션 식별을 위해 세션 ID를 부여하고 세션 ID는 세션 쿠키를 이용하여 클라이언트와 서버 간에 주고받음
- 상태정보를 서버에 저장하기 때문에 쿠키 방식에 비해 보안상 안전

### 보안 취약점
- HTTP 세션 하이재킹(Session ID Hijacking) : 세션 ID 정보를 탈취하여 정상 사용자로 위장하여 접근하는 것
