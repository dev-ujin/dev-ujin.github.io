## 윈도우 부팅순서

### 윈도우 XP, 윈도우 서버 2000/2003의 부팅 순서
1. POST(Power On Self Test) 실행
2. 기본 부팅 관련 설정사항 로드
3. MBR(Master Boot Record) 로드
4. NTLDR(NT Loader) 실행
5. NTDETECT.com 실행
6. ntoskrnl.exe(NT OS Kernel) 실행

### 윈도우 비스타, 2008, 7의 부팅 순서
1. POST(Power On Self Test) 실행
2. 기본 부팅 관련 설정사항 로드
3. MBR(Master Boot Record) 로드
4. 윈도우 부트 서브 시스템(Window Boot Manager) 실행
5. 윈도우 OS 로더(Winload.exe) 실행

# [2] 윈도우 보안

## 윈도우 계정, 권한, 인증

### 기본 사용자
1. Administrator : 관리자 계정. 사용 가능한 계정 중 가장 강력한 권한을 가짐
2. SYSTEM : 시스템에서 최고 권한을 가진 계정으로 로컬에서 관리자보다 상위 권한을 가짐. 이 계정으로 시스템에 로그인 불가
3. Guest : 매우 제한적인 권한을 가진 계정으로 디폴트는 사용 불능

### 기본 그룹
1. Administrators : 도메인 자원이나 로컬 컴퓨터에 대한 모든 권한이 있음
2. Account Operators : 사용자나 그룹 계정을 관리하는 그룹
3. Backup Operators : 시스템 백업을 위해서 모든 시스템의 파일과 디렉터리에 접근할 수 있음
4. Guests : 도메인 사용 권한이 제한된 그룹으로 시스템의 설정 변경 권한이 없음
5. Print Operators : 도메인 프린터에 접근할 수 있음
6. Power Users : 디렉터리나 네트워크 공유, 공용 프로그램 그룹 생성, 컴퓨터의 시계 설정 권한
7. Replicator : 도메인에 있는 파일을 복제할 수 있는 권한을 가지고 있음
8. Server Operators : 도메인의 서버를 관리할 수 있는 권한을 가진 그룹으로 로컬 로그인과 시스템 재시작 및 종료 권한
9. Users : 도메인과 로컬 컴퓨터를 일반적으로 사용하는 그룹으로 사용자 환경을 직접 만들 수는 있지만 설정에는 한계가 있고, 시스템의 시작 및 종료 권한이 없으며, 디렉터리 공유 설정을 할 수 없음

### SID(Security Identifier)
- 윈도우의 각 사용자나 그룹에 부여되는 고유한 식별번호
- 명령어 : whoami/user
- RID(Relative ID) : 관리자는 500번, 게스트는 501번, 일반 사용자는 1000번 이상의 숫자

## 공유자료 관리

### 네트워크 드라이브의 이해
- 다른 컴퓨터의 드라이브를 내 컴퓨터에서 드라이브처럼 사용할 수 있는 기능
- 설정 방법
```java
net use 드라이브명: \\ip\설정대상드라이브$
```

### 디렉터리 및 파일에 대한 접근 권한 규칙
1. NTFS 접근 권한은 누적됨
2. 파일에 대한 접근 권한이 디렉터리에 대한 접근 권한보다 우선함
3. 허용보다 거부가 우선함

### 공유 폴더 숨기기
- 공유 이름 끝에 $를 붙이면 리소스를 공유하면서도 숨길 수 있음
- 숨겨진 공유 폴더 보는 명령어 : net share

## 암호 기능 사용

### 폴더 및 파일 암호화(EFS)
- 윈도우 운영체제에서 개별 파일 또는 특정 폴더 안에 들어 있는 모든 파일을 암호화할 수 있는 기능을 제공

### 볼륨 암호화(BitLocker)
- 윈도우 운영체제에서 제공하는 볼륨 단위의 데이터 암호화 기능
- 볼륨 : 논리적 드라이브(파티션 드라이브)
- BitLocker가 활성화되면 컴퓨터를 시작하는데 필요한 시스템 파티션 부분을 제외하고  해당 볼륨에 저장된 모든 파일이 자동으로 암호화됨

## 레지스트리(Registry) 활용

### 개념
- 윈도우 시스템이 운영되는데 필요한 거의 모든 정보를 담고 있음

### 레지스트리 편집기
- 실행하기 위해서는 [시작]메뉴의 검색란에 [regedit] 혹은 [regedt32]를 입력
- 레지스트리는 윈도우 부팅 시 하이브 파일에서 값을 읽어 들여 구성
- Master Key : 하이브 파일에서 직접 읽어 들여 구성되는 키
  - HKLM(HKEY_LOCAL_MACHINE), HKU(HKEY_USERS)
- Derived Key : Master Key로부터 값을 가져와서 재구성하는 키
  - HKCU(HKEY_CURRENT_USER), HKCC(HKEY_CURRENT_CONFIG), HKCR(HKEY_CLASSES_ROOT)

### 레지스트리 루트키
- HKEY_CLASSES_ROOT(HKCR) : 시스템에 등록된 파일 확장자와 그것을 열 때 사용할 에플리케이션에 대한 맵핑 정보, COM(Component Object Model) 오브젝트 등록 정보를 저장하고 있음
- HKEY_CURRENT_USER(HKCU) : 현재 시스템에 로그인하고 있는 사용자와 관련된 시스템 정보를 저장하고 있음
- HKEY_LOCAL_MACHINE(HKLM) : 컴퓨터에 설치된 하드웨어와 하드웨어를 구동하는 데 필요한 드라이버나 설정 사항에 관련된 정보를 갖고 있음
  - HKLM/HARDWARE : 파일이 아닌 메모리에 휘발성 정보로만 존재
  - HKLM/SAM : 사용자의 로컬 계정 정보와 그룹 정보를 가지고 있음. 시스템 계정 이외에는 접근이 불가능
  - HKLM/SECURITY : 시스템 범위의 보안 정책과 사용자 권리 할당 정보. 시스템 계정 이외에는 접근이 불가능
  - HKLM/SOFTWARE : 시스템 범위의 소프트웨어 목록과 그 환경 설정 정보
  - HKLM/SYSTEM : 시스템이 부팅될 때 필요한 시스템 범위의 환경 설정 정보
- HKEY_USERS(HKU) : 시스템에 있는 모든 계정과 그룹에 관한 정보로 user.dat에 그 내용을 저장
- HKEY_CURRENT_CONFIG(HKCC) : 시스템이 시작할 때 사용하는 하드웨어 프로파일 정보

### 레지스트리 보호
- 레지스트리 접근 제한 : 사용자 계정 컨트롤(UAC, User Account Control)에 의해 일반 사용자는 관리자 암호를 모르면 레지스트리 편집기를 사용할 수 없음
- 현재 레지스트리 상태 저장 : 시스템의 현재 상태를 스냅샷을 찍어 복사해두면 무언가 잘못되었을 때 시스템 복원 프로그램이 이전으로 되돌려 줌
- 레지스트리 키 디스크에 복사 : 키 데이터를 레지스트리 등록 파일(확장자가 .reg인 텍스트파일)에 저장

### 레지스트리 공격
- 부팅 시 악성코드 실행
  1. 개별사용자 지속용 : HKCU\Software\7Microsoft\Windows\CurrentVersion\Run
  2. 개별사용자 일회용 : HKCU\Software\7Microsoft\Windows\CurrentVersion\RunOnce
  3. 전체사용자 지속용 : HKLM\Software\7Microsoft\Windows\CurrentVersion\Run
  4. 전체사용자 일회용 : HKLM\Software\7Microsoft\Windows\CurrentVersion\RunOnce
- 특정 확장자 실행 시 악성코드 실행 : HKEY_CLASSES_ROOT 위치의 값을 공격자가 임의로 조작하여 특정 확장자 실행 시에 원하는 악성 프로그램을 실행

## 윈도우 방화벽(PC 방화벽) 설정

### 개념
- PC 내부로 유입되는 패킷뿐만 아니라 나가는 패킷까지 모두 차단하고 사용자에게 해당 네트워크 패킷의 적절성 여부를 확인하여 공격자로부터 PC를 보호하기 위해서 사용

### 설정 방법
- [제어판]-[관리도구]-[서버관리자]의 [구성]-[고급 보안이 설정된 Windows 방화벽]에서 인바운드/아웃바운드 규칙을 통해 설정할 수 있음

# [3] 윈도우 서버 보안

## 계정 관리

### Administrator 계정 이름 바꾸기
- Administrator 계정의 로그인 시도 실패 횟수의 제한이 없는 점을 이용해 악의적인 사용자가 패스워드 유추 공격을 계속해서 시도할 수 있음

### Guest 계정 상태
- Guest 계정이 불필요한 경우에는 비활성화를 시켜 외부자가 시스템의 정보를 확인하거나 관리자 그룹에 추가하는 것을 막음

### 계정 잠금 임계값과 잠금기간 설정
- 계정 잠금 임계값을 설정하여 로그인 실패 횟수를 제한
- 계정 잠금 기간을 설정하면 지정한 기간 동안 잠긴 계정은 사용 불가

### 관리자 그룹에 최소한의 사용자 포함
- 관리 업무를 위한 계정과 일반 업무를 위한 계정을 분리하여 사용
- 관리자 그룹에는 가능한 최소한의 사용자만 포함하도록 하여야 함

### 암호와 패스워드 정책 설정
- 패스워드는 복잡성을 만족하도록 설정
- 최소 암호 길이는 8문자, 최대 사용기간은 90일, 최소 사용기간은 1일 또는 1주

## 서비스 관리

### 공유 권한 및 사용자 그룹 설정
- 디폴트 공유인 C$, D$, Admin$, IPC$ 등을 제외한 공유 폴더가 Everyone 그룹으로 공유가 금지되었는지를 점검

### 불필요한 공유 제거
- C$, D$, Admin$와 같은 기본 공유는 관리상의 목적으로 사용하지 않는다면 제거해야 함
- IPC$는 제거할 경우 특정 서비스가 실행되지 않을 수 있으므로 제거하지 않음
- 공유 제거 방법
  1. 명령어  
    ```java
    net share 공유이름 /delete
    ```
  2. 레지스트리 값을 0으로 수정(키 값이 없을 경우 새로 생성함)

### Null Session 접근 차단 설정
- 비인가된 사용자가 Null Session을 통해 사용자 인증을 거치지 않고 서버에 접근하여 시스템 내부로 접근이 가능한 취약점이 존재
- 방화벽, 라우터, 네트워크 고급설정 등에서 원천적으로 차단 가능

### 불필요한 서비스 제거
- 시스템에 필요하지 않은 취약한 서비스들이 디폴트로 설치되어 실행되고 있으며 이런 서비스는 공격 지점이 될 수 있으므로 필요하지 않은 서비스의 실행파일은 사용하지 않거나 제거해야 함
- 취약한 서비스들
  - Alerter : 서버에서 클라이언트로 경고 메시지를 보냄
  - Clipbook : 서버 내 Clipbook을 다른 클라이언트와 공유
  - Messenger : net send 명령어를 이요하여 클라이언트에 메시지 보냄
  - Simple TCP/IP Services(Echo, Discard, Character Generator, Daytime, Quote of the Day)

### Anonymous FTP 금지
- 기본적인 FTP 서비스는 계정과 패스워드가 암호화되지 ㅇ낳은 채 전송되어 스니핑이 가능하므로 FTP 서비스를 사용하지 않을 것을 권고
- 불가피하게 사용해야하는 상황이라면 FTP Default 설정이 익명연결 허용을 차단하고, 특정 IP 주소에서 접속하도록 접근제어 설정을 적용

## 패치 관리

### 감사 정책에 따른 시스템 로깅 설정
- 감사 설정 수준이 너무 낮으면 보안 관련 문제 발생 시 원인을 파악하기 어려움
- 감사 설정 수준이 너무 높으면 보안 로그에 불필요한 항목이 많이 기록되어 시스템 성능에 영향을 줄 수 있음

## 보안 관리

### SAM(Security Account Manager) 파일 접근 통제 설정
- SAM 파일은 사용자와 그룹 계정의 패스워드를 관리하고, LSA(Local Security Authority)를 통한 인증을 제공
- Administrators 및 System 그룹 외에는 SAM 파일 접근을 제한하여야 함

### 화면 보호기 설정
- 사용자가 자리를 비운 사이 임의의 사용자가 시스템에 접근하여 악의적인 행위를 하는 것을 막기 위해 자동으로 로그오프 되거나 워크스테이션이 잠기도록 설정하여야 함

# [4] 윈도우 네트워크 서비스

## 디렉터리 데이터베이스(Directory Database)

### 네트워크 디렉터리 서비스
- 사용자에게 리소스를 투명하게 제공(사용자가 리소스의 정확한 위치와 중간 단계들을 알 필요가 없다는 것을 의미)
- 예 : LDAP(Lightweight Directory Access Protocol), NDS(Novell NetWare Directory Service), Microsoft Active Directory

### Active Directory
- 네트워크상의 객체에 대한 정보를 저장하며 관리자와 사용자가 이 정보를 쉽게 찾아 사용할 수 있도록 함

### Active Directory 서버
- 도메인 내에서 서버로 동작하는 컴퓨터는 구성원 서버나 도메인 컨트롤러의 역할 중 하나를 수행
- 도메인 내에 있지 않은 서버는 독립 실행형 서버가 됨

### DC(Domain Controller)와 AD(Active Directory)
- Windows 환경에서 사용자가 로그인하면 사용자는 데이터베이스에 계층적 디렉터리를 가지고 있는 **DC**에 로그인함
- 이 데이터베이스는 네트워크 리소스를 정리하고 사용자 접근 통제 기능을 수행하는 디렉터리 서비스인 **AD**를 실행함
